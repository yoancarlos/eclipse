Understanding the Integration Content Types

SAP HANA Cloud Integration offers Eclipse-based tooling that provides features to model integration flows, configure with attributes and deploy them to the runtime.

The Integration tooling works in the local development mode in which you create an Integration Project in your local Eclipse workspace and start developing integration content using the features available in the Integration Designer perspective. Once the content is ready, you deploy the project to the runtime.

Types of Integration Projects

The tooling offers you different project types on the basis of objects involved, one is the integration flow project and the other is the value mapping project.

Integration Flow Project Type

The integration project types contain the integration flow configurations. You can deploy the integration projects either to an IFL or an IFLMAP runtime node.

Integration Project Structure
Elements in Project Structure Description
src.main.resources.mapping Folder for mappings to be used in scenario
src.main.resources.scenarioflows.integrationflow Folder for BPMN integration flow
src.main.resources.wsdl Folder for interfaces like IDOC , WSDL used in scenario
MANIFEST.MF File contains dependencies to runtime components
Note Additional files that you can add to the integration project are:
Additional Elements in Project Structure Description
parameters.prop File contains externalized attributes representing a variable like landscape data
parameters.propdef File contains metadata such as value type of the parameterized attribute, its description and whether the attribute should be configured mandatorily.
Value Mapping Project Type

The Value Mapping project type is used for scenarios that require you to map different representations of an object to each other. Each value mapping project contains one or more value mapping group that is a set of values of the object.You can deploy the value mapping projects either to an IFL or an IFLMAP runtime node.

Value Mapping Project Structure
Elements in Project Structure Description
MANIFEST.MF File contains dependencies to runtime components
value_mapping.XML File contains value mapping groups that hold the objects and their corresponding representations
FSN Mapping Project Type

The FSN Mapping project type is used to implement scenarios that require mapping belonging to the Financial Services Network. The project mainly contains a mapping package in which you can place one or more mapping files. The mapping files have extension .vpjzip and contains one or more message transformation definitions. You have to deploy the FSN Mapping project type on an IFL runtime node.

FSN Mapping Project Structure
Elements in Project Structure Description
src.main.resources.mapping Folder contains mapping to be used for an FSN scenario.
MANIFEST.MF File contains dependencies to runtime components.
