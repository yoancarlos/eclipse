Installing Eclipse and Dependent Plug-Ins

You install the features of SAP HANA Cloud Integration on the Eclipse integrated development environment (IDE) to access the configuration and administration functions.

Context

The Integration Designer feature can be used with the following Eclipse version: Kepler release (Eclipse 4.3).

Procedure

  1. Create a folder in your file system with a name representing the customer tenant. For example, C:\eclipse\ide\<customer_tenant>... .
    This folder path would help you to maintain a customer-specific workspace.
  2. Download the relevant Eclipse IDE.
    Option Description
    To use Eclipse Kepler edition Download Eclipse Standard 4.3.2 IDE from the list available at http://www.eclipse.org/downloads/packages/release/kepler/sr2.
  3. Extract the content of the downloaded eclipse-... .zip file to a folder.
  4. Double-click the eclipse.exe file to launch the Eclipse IDE.
  5. In the Workspace Launcher dialog, select the folder path you have created above in the Workspace field.
  6. Choose OK.
  7. Choose  Window  Preferences   General  Network Connections .
  8. On the Network Connections page, select the Native option from the Active Provider dropdown.
  9. Choose Apply and then choose OK.
  10. Choose  File   Restart  to restart the Eclipse IDE.
  11. From the main menu, choose  Help  Install New Software... .
  12. On the Available Software page of the Install wizard, choose Add... next to the Work with dropdown.
  13. In the Add Repository dialog, in the Location field, enter http://nexus:8081/nexus/content/repositories/build.snapshots.unzip/com/sap/ifl/com.sap.it.ide.ifl.kepler.site/RELEASE/com.sap.it.ide.ifl.kepler.site-RELEASE-assembly.zip-unzip/, and choose OK.
    Note that this Eclipse Update Site link points you always to the newest internally released version of the Integration Designer and Integration Operations tools.

    It is important that for the Integration Designer and Integration Operations tools you use the same version as for the SAP HCI software (runtime components. When you install the newest internally released tool version (using the internal Eclipse Update Site above), you will get a newer version compared with the the at the time (externally) released SAP HCI software version (which most likely you use). From that point in time, your tooling will not work anymore correctly together with the (now older) SAP HCI software.

    To overcome this issue, check for the What's New topic that corresponds to the actual released SAP HCI software.

    There is also a snapshot update link for testing purposes.

    As snapshot link use:

    http://nexus.wdf.sap.corp:8081/nexus/content/repositories/build.snapshots.unzip/com/sap/ifl/com.sap.it.ide.ifl.kepler.site/1.17.0-SNAPSHOT/com.sap.it.ide.ifl.kepler.site-1.17.0-SNAPSHOT-assembly.zip-unzip/

    Note that the snapshot link currently is release-dependent. It contains the version number.

  14. In the Name field, enter a descriptive name for the update site and choose OK.
  15. On the Available Software page of Install wizard, select the following three options:
    • Show only the latest version of available software
    • Groups items by category
    • Contact all update sites during install to find required software
  16. In the section of the page where the list of features are displayed, select the HANA Cloud Integration Tools category.
    Selecting the HANA Cloud Integration Tools category automatically selects the Designer for SAP HANA Cloud Integration and Operations for SAP HANA Cloud Integration features.
  17. Choose Next.
  18. On the Install Details page, check the features to be installed and choose Next.
  19. On the Review Licenses page, accept the terms of licensing agreement and choose Finish.
  20. In the Software Updates dialog, choose Yes to restart the Eclipse IDE.

Results

These steps ensure that the software features of SAP HANA Cloud Integration are now installed.

Next Steps

Once the Eclipse IDE launches, obtain the URL of the Operations Server and the SCN user credentials from the SaaS Admin, and enter the details at  Window  Preferences  SAP HANA Cloud Integration  Operations Server . This setting is mandatory if you are deploying integration flows.
