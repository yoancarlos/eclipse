SAP Integration Flow Templates

When you model and configure an integration flow, the tool validates the integration flow with the closest matching SAP supported scenario. If the integration flow configuration does not match the SAP supported modelling, the tool shows you a list of allowed scenarios in the Console. Contact SAP Support if you require additional integration scenarios to be supported.

Note The section below suggests the valid elements for each integration scenario, but the elements can be used only in a specific sequence.

Integrate with an external SOAP Endpoint

Integration between sender SOAP (SOAP 1.x) connector and receiver SOAP (SOAP 1.x) connector
Model your integration flow using a combination of the elements given below, and configure the sender channel with SOAP (SOAP 1.x) connector and the receiver channel with SOAP (SOAP 1.x) connector.
  • Point-to-Point: If the sender participant needs a simple message exchange with a receiver participant, use a Point-to-Point integration flow.
  • (Optional)Gateway: If the sender participant needs to forward selected messages to a particular receiver based on certain condition, use a gateway and configure condition for the routing branch.
  • Persist Message: If you want to store the message payload at a particular step, use persist message.
  • Message Transformation Elements:
    • (Optional)Mapping: If the sender participant can communicate with the receiver participant only after the message payload is transformed in a specific format, use a mapping between the sender and receiver participants
    • (Optional)Enricher: If the message should be provided with additional information to the receiver, use an enricher.
    • (Optional)Filter: If a specific content needs to be extracted from the message, use a filter.
    • Optional)Decoder: If the message should be decoded using Base64, use an decoder.
  • Security Elements:
    • (Optional) XML Digital Signer with Enveloping Signature: If the XMl digital signature should contain the content to be signed within itself, use an XML digital signer of type enveloping signature.
    • (Optional) XML Digital Signer with Enveloped Signature: If the XML digital signature needs to be inserted within the content to be signed, use an XML digital signer of type enveloped signature.
    • (Optional)XML Signature Verifier: If the digital signature of an incoming XML message needs to be verified, use XML signature verifier.
    • (Optional)Decryptor: If the encrypted content in the message body needs to be decrypted, use an encryptor.
