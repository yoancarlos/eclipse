Generating SSH Key for Authentication

Context

You perform the tasks below to authenticate your identity in Gerrit and to be permitted to commit changes into the Git Repository.
Note

This is a single-time activity for a user who needs to clone Git Repository irrespective of tenants and customers.

Procedure

Verify your identity in Gerrit

  1. Click https://git.wdf.sap.corp:8080 to open Gerrit in your web browser.
  2. Select the Sign In option on the top-right corner of the page and enter your SAP_ALL username and password.
  3. Select the Settings option on the top-right corner of the page.
  4. From the left pane, choose Identities.
  5. Check if the e-mail address and user name is displayed here.

Create new entries in the Eclipse Preferences for a GIT user

  1. Launch the Eclipse IDE.
  2. Choose  Window  Preferences .
  3. In the Preferences dialog, select  Team   Git   Configuration  from the left pane.
  4. On the Configuration page, select the User Settings tab.
  5. In the Location field, check if the location appears as <HOME>/.gitconfig.
    Note

    By default, <HOME> is configured to C:\Users\<SAP_ALL ID. Use this default location. In case, your HOME is set to a different location change the environment variables as mentioned in the task Setting the Environment Variables.

  6. Choose New Entry....
  7. In the Add a configuration entry dialog, specify user.name as the Key and your full name as the Value.
    Note The full name should be the same as you checked in the task Verify your identity in Gerrit. Git stores your full name on every push action.
  8. Choose Apply and then choose OK.
  9. In the Add a configuration entry dialog, specify user.email as the Key and your SAP e-mail address as the Value.
    Note

    Git stores your e-mail address on every push action.

Setting the Environment Variables

  1. Right-click My Computer, and select Properties.
  2. Choose the Advanced system settings option from the left pane.
  3. In the Environment Variables dialog, select the $HOME variable in the User variables <SAP_ALL ID> section, and choose Edit….
  4. Enter the location path to be set as your HOME. For example, enter C:\Users\<SAP_ALL ID.

Configure the location of SSH keys

  1. In the Eclipse IDE, choose  Window  Preferences .
  2. In the Preferences dialog,  General  Network Connections   SSH2  from the left pane.
    Note In the General tab, check the SSH2 home value. Make sure it points to <HOME>\.ssh folder.
  3. Select the Key Management tab, and choose Generate RSA key... .
  4. If you want to protect your SSH key, enter a passphrase.
    Note If you choose to use a passphrase, you need to enter it for every interaction with GIT Repository.
  5. Choose Apply and then choose OK.
  6. Choose Save Private Key... to save the generated private/public keys. By default, Eclipse offers the id_rsa as the key name. Use this default name

Upload your SSH public key to Gerrit

  1. Go to <HOME>/.ssh/ folder and open the id_rsa.pub in a notepad.
  2. Copy content to clipboard, which is the public key, from the notepad
  3. Log on to Gerrit and select the Settings option on the top-right corner of the page.
  4. From the left pane, choose SSH Public Keys.
  5. If you are logging on for the first time, paste the content in the Add SSH Public Key text area and choose Add. For subsequent addition of keys, choose Add Key..., then paste the content in the Add SSH Public Key text area and choose Add.
