Defining Message Security

Ensuring security is important, during message exchange, to preserve the integrity of messages. A sender participant uses signatures to ensure authenticity of the sender, and encryption to prevent non-repudiation of messages during the message exchange.

You can sign a message content depending on the format you require:
  • PKCS#7/CMS can be used to protect any message content with or without tags.

    For more information, see Signing the Message Content with PKCS#7/CMS Signature.

  • XML digital signature can be used only for XML messages.

    For more information, see Signing the Message Content with XML Digital Signature.

For more information on how to encrypt the message content, see Encrypting the Message Content.

A receiver participant uses a signature verifier to identify the authenticity of the sender and a decryptor to decrypt the message received over cloud.

The corresponding verifiers for the message content are:
  • PKCS#7/CMS Signature Verifier

    Configuring the PKCS#7/CMS Signature Verifier.

  • XML Digital Signature Verifier

    Configuring XML Digital Signature Verifier.

For more information on how to decrypt the received message content, see Decrypting the Message Content.
