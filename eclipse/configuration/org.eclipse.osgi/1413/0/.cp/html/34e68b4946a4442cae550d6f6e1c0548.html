Testing Integration Flow Project

Prerequisites

You have configured the runtime target platforms and common testing libraries, which are used to execute tests.

Context

You perform this task to execute tests on an integration flow project.

Procedure

  1. In the Project Explorer view, right-click on the project and choose Add Test Case.
  2. In the Add Test Case dialog, enter a test case name to be displayed and select the product profile you have set in the steps above that applies for this test case.
  3. Choose Finish.
  4. In the Project Explorer view, expand the testing folder under the current integration flow project.
  5. Under test.configuration package, right-click on the <Test case name>.ifltest and choose Add Test Method from the context menu.
  6. In the Add Test Method dialog, enter a name for the new test method.
  7. Add an input XML file and an output XML file.
  8. Choose Finish.
  9. If you want to select a particular runtime target platform on which the test should be executed, then perform the following substeps:
    1. From the context menu of the project, select Set Target Platform.
    2. In the Set Target Platform dialog, select the runtime target plaform.from the Runtime dropdown.
    3. If the required runtime target platform is not available in the dropdown list, choose Add....
    Note
    • You can select target runtime platform for multiple integration projects at a time.
    • For more information on how to set the runtime target platform, see Setting the Runtime Platform and Libraries for Testing.
  10. In the Project Explorer view, right-click on the project and choose Execute Tests.
    Note If you open the generated .java file under test.content package, you can find the tests are added as methods. The input and output files are available in test.data package.
    Tip: You can hide the libraries that are displayed under the current integration flow project. Choose the dropdown icon from the toolbar of the Project Explorer view and select  Customize View…  Libraries from External... .
  11. In the JUnit view, check the test name and the status of the tests. If the JUnit view does not show any data, then increase the value for the field Duration for executing test(sec) in the Test Configuration page in Preferences. The default time interval is 8 seconds.
    Restriction
    • If you are executing a test case that involves non-readable characters in the <output>.xml, then the test execution fails. For example, the signature file obtained from PKCS#7/CMS Signature-Content Splitter contains non-readable signed data that causes the test case to fail.
    • If you are executing a test case that involves multiple headers, then the test exeuction can fail.
