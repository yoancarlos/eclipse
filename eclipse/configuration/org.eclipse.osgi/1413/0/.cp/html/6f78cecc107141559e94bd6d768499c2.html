Setting the Runtime Platform and Libraries for Testing

Context

You perform the task to allow the runtime target platform and common testing libraries to be available during the testing of integration flow configuration.

Procedure

  1. Defining Runtime Platforms
    1. In the main menu, select Window > Preferences > SAP HANA Cloud Integration > Testing Configuration > Runtime Platform.
    2. In the Runtime Platform page, choose Add....
    3. In the Add Location dialog, enter the URL of the update site for runtime jars.
      Note
      • If you add multiple runtime platform update sites, then make sure that at least one update site is always selected.
      • If you have not specified any runtime platform update site, a default update site maintained by SAP is automatically triggered.
      • If the currently set runtime update site is invalid, a default update site maintained by SAP is automatically triggered.
      The selected update site is considered by default during test execution.
    4. If the runtime update site has recently been changed, choose Reload.
    5. Choose Apply and then OK.
  2. (optional) Defining the Testing Libraries
    Note You perform this step only if you want to add a specific common testing library.
    1. In the main menu, select  Window  Preferences   SAP HANA Cloud Integration  Testing Configuration  Testing Libraries .
    2. In the Testing Libraries page, choose Add… to enter the URL of the update site of libraries that are associated with the runtime platform but are not available with the framework.
      Note If you add multiple testing libraries update sites , then make sure that at least one update site is always selected as default. If you execute a test configuration without specifying any update site, a default update site for testing libraries is used during execution and it is automatically added to this page .
