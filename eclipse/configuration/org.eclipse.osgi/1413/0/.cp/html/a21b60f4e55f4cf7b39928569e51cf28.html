Merging the Content using EGit

Context

You perform this task to make the content of the <Source> branch of the central Git Repository available in the <Target> branch of the cloned repository.

For example, in a scenario, the content needs to be merged from <TenantID>_Test branch of the Git Repository with the <TenantID>_Live branch of the cloned repository.

Note Merge fails if any file in the Staged Changes or Unstaged Changes has been modified in the <Source> branch. You can either commit the changes or stash it.

Procedure

  1. In the main toolbar, choose  Window  Open Perspective  Others... .
  2. In the Open Perspectivedialog, select Git Repository Exploring.
  3. To create a new <Target> branch into which the content of the <Source> branch is merged, perform the following substeps:
    1. In the Git Repositories view, expand all the nodes under the cloned repository.
    2. Right-click on  Branch    Local  and choose  Switch to…  New Branch  from the context menu.
    3. In the Create Branch wizard, from the Source ref: dropdown, select the branch in the Git Repository that you want to map in the local branch. The Branch name field gets automatically filled with the same name. For example, if the Source ref: branch is refs/head/Live then the branch name is filled as Live.
    4. Under the Pull Strategy section, select the Merge option.
    5. Choose Finish.
    Note The new Target branch gets automatically checked out.
  4. If you already have the <Target> branch under Local and it is not checked out then, right-click on Branches and choose  Switch To...   <Target>Branch .
  5. Right-click on the cloned repository and choose Pull to be updated with any new content.
  6. In the Git Repositories view, right-click on the <Target> branch and choose Merge.
  7. In the Merge dialog, under the Remote section, choose the <Source> branch from which you need to move the content to the <Target> branch.
  8. Choose Merge. The content is now available in the <Target> branch of the cloned repository.
