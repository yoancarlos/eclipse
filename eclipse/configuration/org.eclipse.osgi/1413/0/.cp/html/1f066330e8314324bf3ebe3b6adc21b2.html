Configure Adapter in Communication Channels

You should configure the adapters assigned to communication channels before deploying the integration flow.

The following adapter types are available:

Table 1: Adapter

Adapter

Description

Ariba sender or receiver adapter

Connect a tenant to the Ariba network (this allows SAP and non-SAP cloud applications to send and receive business-specific documents in cXML format to and from the Ariba network).

Facebook receiver adapter

Access and extract information from Facebook based on certain criteria such as keywords or user data (transport protocol: HTTP).

HTTP receiver adapter

Connect to a receiver system using HTTP (transport protocol: HTTP).

HTTPS sender adapter

Connect to a sender system using HTTPS (transport protocol: HTTP).

IDoc sender or receiver adapter

Exchange IDoc messages with another system (transport protocol: HTTP).

LDAP receiver adapter

Connect to LDAP directory service.

Mail sender adapter

Receiver e-mails from an e-mail server (transport protocol: SMTP).

Mail receiver adapter

Send e-mails to an e-mail server (transport protocol: SMTP).

OData sender and receiver adapter

Connect to systems exposing OData services (OData service providers) (transport protocol: HTTP).

OData stands for the Open Data Protocol.

ODC receiver adapter

Connect to SAP Gateway OData Channel

SFTP sender adapter

Connect to a remote system using the SSH File Transfer protocol (also referred to as Secure File Transfer protocol) to read (poll) files from the system (transport protocol: SFTP).

SFTP receiver adapter

Connect to a remote system using the SSH File Transfer protocol to write files to the system (transport protocol: SFTP).

SOAP /SOAP 1.x sender and receiver adapter

Exchange messages with another system that supports SOAP 1.1 or SOAP 1.2 (transport protocol: HTTP).

SOAP / SOAP RM sender and receiver adapter

Exchange messages with another system based on the SOAP communication protocol and SAP RM as the message protocol (transport protocol: HTTP).

SuccessFactors sender and receiver adapter

Connect to a SuccessFactors system using the SOAP, OData, and REST message protocol (transport protocol: HTTP).

Twitter receiver adapter

Access Twitter and either read or post tweets (transport protocol: HTTP).

XI sender and receiver adapter

Connect a tenant to a remote system that can process the XI message protocol.

If you are quick configuring an integration flow, you can configure a few adapter parameters. Refer to the relevant adapter configuration details for information on those parameters.

If you are developing an OData service, you can configure a SOAP, OData or HTTP adapter assigned to the receiver channel and the OData adapter assigned to the sender channel.
