Submitting Content to the Git Repository

Context

Procedure

  1. Share new projects after deploy If you create a new project, you have to share the project to the Git Repository.
    1. In the Project Explorer, right-click on the project and choose  Team  Share Project…  from the context menu.
    2. In the Share Project wizard, select Git as the repository type and choose Next.
    3. In the Configure Git Repository page, from the Repository dropdown, select the Git Repository to which you need to share the project.
    4. Choose Finish. Now your project is available in the Git Repository. In the Git Staging view, under the Unstaged Changes, you can view the list of files of the project.
  2. Commit the changes
    1. In the Git Repositories view, right-click on the project and choose Pull from the context menu to avoid any conflicts while pushing the content.
      Note It is always advised to pull the content before you commit your changes to merge any conflicting changes.
    2. Open the Git Staging tab view.
    3. If any file is listed under the Unstaged Files section, right-click on the file and choose Add to index from the context menu. This moves the files to the Staged Files area.
    4. From the toolbar of the Git Staging view, choose (Add Signed-off by icon) and then choose (Add Change-Id icon). You can also choose to write a description of the commit action in the first line of the Commit MessageCommit Message section.
    5. In the toolbar of the Git Staging view, choose (Commit icon). The files are committed locally.
  3. Push content to the Git Repository
    1. Under  Remotes  origin , right-click on the URL of the remote repository indicated with (Push icon).
    2. Choose Configure Push...
    3. Expand Advanced section and choose Edit(Advanced)… to change reference specifications.
    4. In Push Ref Specification wizard, select refs/heads/<working branch in cloned repository> as the Source ref: and refs/heads/<branch in Git Repository> as the Destination ref:. For example, select refs/heads/<TenantID>_Test as the Source ref: and refs/heads/<TenantID>_Test as the Destination ref:.
    5. Choose Add Spec+.
    6. Choose Finish.
    7. Choose Save and Finish. Push Result dialog shows you the content that is pushed into the central Git Repository.
    In case you are unable to push the changes as the changes are being rejected, click https://git.wdf.sap.corp:8080/Documentation/error-messages.html, and follow the available troubleshooting information.
