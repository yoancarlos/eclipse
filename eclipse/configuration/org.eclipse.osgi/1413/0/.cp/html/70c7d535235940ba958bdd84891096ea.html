Pushing Content to the Git Repository

Context

You perform this task to push the developed content into the Live branch of the customer Git repository.

Procedure

  1. In the Git Repositories view, right-click on the project and choose Pull from the context menu to pull any changes that other users have recently pushed.
  2. Under  Remote  origin , right-click on the URL of the remote repository indicated with (Push icon).
  3. Choose Configure Push.
  4. If you want to change reference specifications, expand Advanced.
  5. In Push Ref Specification wizard, select refs/heads/<Target branch of cloned repository> as the Source ref: and refs/heads/<Target branch of central Git Repository> as the Destination ref:.

    Suppose the content needs to be pushed from the <TenantID>_Live branch of the cloned repository to the <TenantID>_Live of the central Git Repository.

     

    So, in the Push Ref Specification wizard, select refs/heads/<Tenant>_Live as the Source ref: and refs/heads/<Tenant>_Live as the Destination ref:.

  6. Choose Add Spec+.
  7. Choose Finish.

    Push Result dialog shows you the content that is pushed into the central Git Repository. Now the customer specific content is available in the Live branch of the Git Repository.
